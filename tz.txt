🎓 Imtihon topshirig‘i: Online Kurslarni Boshqarish Tizimi

✅ Maqsad
Siz mustaqil ravishda NestJS asosida Online Kurslarni Boshqarish Tizimi backend qismini ishlab chiqasiz. Ushbu tizim foydalanuvchilarga kurslarga yozilish, mashg‘ulotlarni ko‘rish va topshiriqlarni bajarish imkoniyatini beradi. Admin esa kurslarni boshqaradi.

🕒 Umumiy vaqt: 4 kun

🗂️ Texnik talablar
1️⃣ Texnologiyalar
Backend: NestJS (TypeScript bilan)


Ma’lumotlar ombori: PostgreSQL


ORM: TypeORM yoki Prisma


Autentifikatsiya: JWT


Konteynerizatsiya: Docker, docker-compose


Versiya nazorati: Git + GitHub


API test: Postman



2️⃣ Asosiy funksiyalar
📌 1-kun (Asosiy tayyorgarlik va autentifikatsiya)
NestJS loyihasini yaratish (TypeScript bilan)


PostgreSQL bilan bog‘lash (docker-compose tavsiya etiladi)


Auth modulini yaratish:


POST /auth/register — Foydalanuvchi (student) ro‘yxatdan o‘tadi.


POST /auth/login — Foydalanuvchi tizimga kiradi.(2 ta token access and refresh ) refresh cookieda saqlansin)
POST /auth/refresh - tokenni yangilash 


JWT token orqali autentifikatsiya.


User Entity/Schema:


name, email (unikal), password (hashlangan), role (student yoki admin)



📌 2-kun (Kurslar boshqaruvi)
Courses modulini yaratish:


GET /courses — Barcha kurslarni ko‘rish.


POST /courses — Yangi kurs qo‘shish (faqat admin).


PUT /courses/:id — Kursni o‘zgartirish (faqat admin).


DELETE /courses/:id — Kursni o‘chirish (faqat admin).


Course Entity/Schema:


name, description, price, teacher, category, level



📌 3-kun (Modullar va darslar, kursga yozilish)
Modules va Lessons modullarini yaratish:


GET /courses/:courseId/modules — Kursdagi modullarni ko‘rish.


GET /modules/:moduleId/lessons — Moduldagi darslarni ko‘rish.


Enrollment (Yozilish):


Foydalanuvchi kursga yozilishi.


Yozilgandan keyin modul va darslarga kirish imkoniyati bo‘lsin.


Relationships:


Kurs ➝ Modullar ➝ Darslar



📌 4-kun (Topshiriqlar va natijalar)
Assignments moduli:


POST /modules/:moduleId/assignment — Foydalanuvchi topshiriq topshiradi.


Results moduli:


GET /results — Foydalanuvchi o‘z natijalarini ko‘radi.


Baholash logikasi:


Avtomatik yoki qo‘lda baholash (minimal logika yetarli).



🔑 Texnik talablar (yakuniy)
